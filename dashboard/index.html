Arduino Hardware Guide

Everything you need to wire up, flash, and control a physical hand or arm from KYMA.

Requirements

  • Arduino Uno / Mega / Nano (or any board with 8+ PWM-capable pins)
  • Up to 8 servo motors (SG90 micro servos or MG996R for higher torque)
  • External 5V power supply for servos
  • USB cable to connect Arduino to the PC running KYMA
Do not power servos from the Arduino 5V pin — it cannot supply enough current. Always use an external supply with a shared ground.

Wiring

                  +---------------------+
                  |     Arduino Uno     |
                  |                     |
 Servo 0 <--------| Pin 2         GND --+---- Servo GND (all)
 Servo 1 <--------| Pin 3              |     + Power Supply GND
 Servo 2 <--------| Pin 4              |
 Servo 3 <--------| Pin 5         5V  --+---- (board only)
 Servo 4 <--------| Pin 6              |
 Servo 5 <--------| Pin 7              |
 Servo 6 <--------| Pin 8              |
 Servo 7 <--------| Pin 9              |
                  |                     |
                  |  USB <-- to PC     |
                  +---------------------+

         External 5V PSU
         +--------------+
    5V --+              +-- Servo VCC (all, red wire)
   GND --+              +-- Common GND (shared with Arduino GND)
         +--------------+

Flashing the Firmware

  1. Open firmware/arm_controller/arm_controller.ino in the Arduino IDE
  2. Select your board under Tools > Board
  3. Select the correct COM port under Tools > Port
  4. Click Upload
  5. Servos will move to 90° (home position) on boot

Customizing the Firmware

Edit the configurable arrays at the top of arm_controller.ino:

const uint8_t SERVO_PINS[NUM_JOINTS]  = { 2, 3, 4, 5, 6, 7, 8, 9 };  // which pins
const uint8_t HOME_ANGLES[NUM_JOINTS] = { 90, 90, 90, 90, 90, 90, 90, 90 };  // neutral
const uint8_t MIN_ANGLES[NUM_JOINTS]  = {  0,  0,  0,  0,  0,  0,  0,  0 };  // lower limits
const uint8_t MAX_ANGLES[NUM_JOINTS]  = {180,180,180,180,180,180,180,180 };  // upper limits
Adjust min/max to match your hardware — for example, limit finger servos to 30–150° to avoid mechanical binding.

Binary Protocol

KYMA talks to the Arduino over serial at 115200 baud using a compact binary protocol:

CommandBytes SentResponseDescription
MOVE[0x01, joint, angle][0xAA, joint]Set servo angle (clamped to min/max)
ESTOP[0x02][0xAA, 0x00]Emergency stop — detaches all servos
HOME[0x03][0xAA, 0x00]Return all servos to neutral (90°)
PING[0x04][0xBB, 0x00]Heartbeat check
DWRITE[0x05, pin, value][0xAA, pin]Digital write (LEDs, relays, etc.)
AWRITE[0x06, pin, value][0xAA, pin]Analog/PWM write

Error responses use [0xFF, reason]: 0x01 = timeout, 0x02 = ESTOP active, 0x03 = invalid joint.

Exporting .ino from the Block Editor

The Block Editor can generate standalone Arduino sketches that run without the KYMA server:

  1. Go to the Blocks tab and build your program with servo moves, delays, loops, and conditionals
  2. Click Export .ino in the toolbar to preview the generated code
  3. Click Download .ino to save the file
  4. Open it in Arduino IDE and upload to your board

The generated sketch includes Servo.h, pin attachments, and your block sequence translated to C++. Supports: servo writes, sweeps, delays, digital/analog output, loops, conditionals, and serial logging.

Blocks that depend on live EMG data (like If RMS >) are translated to analogRead() calls. This works if you wire EMG amplifier output directly to Arduino analog pins, but won't have the same filtering as the full KYMA pipeline.

Connecting to KYMA

Once the firmware is flashed, start the server with your Arduino port:

# Windows PowerShell:
$env:ARDUINO_PORT="COM4"; $env:EMG_MOCK="1"; python main.py

# Linux/macOS:
ARDUINO_PORT=/dev/ttyUSB1 EMG_MOCK=1 python main.py

The server auto-detects the Arduino on startup (looks for the [0xBB, 0xFF] ready signal). Check the dashboard status bar to confirm the connection.
